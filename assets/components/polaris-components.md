# Polaris Components

Use these patterns for merchant tools, commerce operations, and storefront administration.

## Page Scaffold

- Context header with store or channel state
- Utility-first toolbar for filtering and actions
- Main list, form, or order/content region

## Component Priorities

- Primary: resource list, form layout, card, banner, button, modal, index table
- Supporting: badges, help text, bulk actions, empty states

## Behavior Rules

- Keep merchant tasks efficient and reassuring
- Use clear error/help copy
- Favor approachable structure over cold enterprise density
- Surface business state clearly: active, draft, paid, unavailable

## Tailwind Notes

- Primary: `#008060`
- Soft card surfaces and moderate radii
- Balanced spacing with friendly density

## Example Pattern

- Order management screen
  - store context header
  - filter and sort controls
  - resource table
  - side panel or modal for edits
