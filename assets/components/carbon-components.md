# Carbon Components

Use these patterns when enterprise rigor, density, accessibility, and structural clarity are the main priorities.

## Page Scaffold

- Strong page title and informational summary
- Structured toolbar with explicit actions
- Dense content regions with strict alignment
- Side panels and inline expansion for secondary detail

## Component Priorities

- Primary: button, text input, select, data table, accordion, side panel, notification
- Supporting: skeleton, tooltip, overflow menu, inline loading, progress indicator

## Behavior Rules

- Favor keyboard-first efficiency
- Use labels and helper text aggressively for clarity
- Keep layouts grid-driven and tightly aligned
- Prefer restrained styling over expressive ornament

## Tailwind Notes

- Typography: IBM Plex Sans and IBM Plex Mono
- Grid: 16-column
- Base spacing: 16px
- Radius: mostly square or minimal rounding
- Primary: `#0F62FE`

## Example Pattern

- Analytics dashboard
  - title and metadata strip
  - KPI row
  - data table with overflow menu
  - side panel for row detail and edit actions
