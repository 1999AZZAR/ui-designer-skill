# Neo-M3 Components

Use these patterns when the product needs both strong product structure and a bolder editorial or tech-forward voice.

## Component Priorities

- Primary: rounded cards with strong borders, technical metadata blocks, bold CTA modules
- Supporting: dashed experimental containers, hard-shadow callouts, mono data chips

## Behavior Rules

- Combine strong geometry with disciplined layout
- Keep the visual voice bold without becoming chaotic
- Use monospace sparingly for metadata and system cues
